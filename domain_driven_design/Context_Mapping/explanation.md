# Context Mapping

## Introduction

Context mapping describes how bounded contexts relate and integrate. It defines ownership, dependencies, and translation paths between teams or systems.

## Common Relationships

- Upstream/Downstream: one context supplies data to another.
- Conformist: downstream adapts to upstream model.
- Partnership: two contexts evolve together.
- Anti-Corruption Layer: translation at the boundary.

## Practical Guidance

- Document context maps early to reduce integration friction.
- Make ownership and integration contracts explicit.
