# Tactical vs Strategic DDD

## Introduction

Tactical DDD is about code-level building blocks. Strategic DDD is about boundaries, language, and team alignment. You often start with strategic clarity before adding tactical patterns.

## Tactical Building Blocks

- Entities, value objects, aggregates
- Repositories and factories
- Domain services and events

## Strategic Building Blocks

- Bounded contexts
- Context mapping
- Ubiquitous language

## Practical Guidance

- Start with strategic DDD when domains are unclear or team boundaries are fuzzy.
- Use tactical patterns only when they reduce complexity in code.
