# Why Laravel

Laravel is a batteries-included PHP framework for building web apps and APIs quickly with strong conventions.

Key reasons teams choose it:
- Productive defaults: routing, validation, ORM, queues, caching, and testing are built in.
- Clear architecture: MVC-ish structure with a strong service container and middleware pipeline.
- Ecosystem: first-party packages (Horizon, Telescope, Sanctum, Octane) and wide community support.
- Developer experience: Artisan CLI, readable syntax, and excellent documentation.

When it is a good fit:
- You want a full-stack framework that ships the common pieces (auth, jobs, caching, mail).
- You need a fast time-to-market with clear conventions.
- You plan to scale with queues, caching, and worker processes.

When it is not a fit:
- You want a minimal micro-framework with almost no conventions.
- You need to avoid the framework layer for extreme latency or footprint constraints.
